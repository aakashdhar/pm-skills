name: refresh-dashboard
category: dashboard
description: >
  Reads all project files and generates dashboard.html using the locked
  BetaCraft template. Brand — logo, colour, name — is read from org.json
  in the project root, so the dashboard shows the correct delivering
  organisation (BetaCraft, Free Range, or any other). Every section
  sourced from actual documents — nothing invented. Triggers on
  "refresh dashboard" or "update the dashboard". Saves dashboard.html
  to project root. Does not commit — PM reviews in browser first,
  then runs publish-dashboard.

# Skill: refresh-dashboard

---

## Trigger

- `"refresh dashboard"`
- `"update the dashboard"`
- `"generate the client dashboard"`
- `"rebuild the dashboard"`

## Pre-read (all required)

1. `org.json` — delivering org name, logo, colour, email ← READ FIRST
2. `project.md` — status, RAG, phase, team, risks
3. `shared/HOTSHEET.md`
4. `shared/DECISIONS.md`
5. `shared/CONFLICTS.md`
6. `reference/sow.md`
7. `requirements/` — all files
8. `deliverables/` — all files
9. `change-log/` — all files
10. `comms/meetings/` — last meeting summary
11. `comms/summary.md`
12. `dashboard-versions/` — for version number
13. `client.md`

---

## org.json — what it controls

```json
{
  "name": "Free Range",
  "logo": "https://freerange.com/hubfs/Free%20Range/Chicken.svg",
  "primary": "#FF4E02",
  "email": "aakash@freerangestudios.com"
}
```

| Field | Where it appears in dashboard |
|---|---|
| `name` | Header brand name · Footer "Generated by [name]" |
| `logo` | Header logo `<img src>` · Footer logo |
| `primary` | All pills · links · RAG badges · timeline fill · today marker |
| `email` | Footer contact |

If `org.json` is missing — fall back to BetaCraft defaults:
```
name:  BetaCraft
logo:  https://betacraft.com/logo.svg
primary: #E22A38
email: aakash@sorigin.com
```

## CSS variable substitution

When generating dashboard.html — replace these root variables:

```css
--primary: [org.primary]
--primary-light: [org.primary at 10% opacity — calculate]
--primary-dark:  [org.primary darkened 15% — calculate]
```

Logo: `<img src="[org.logo]" alt="[org.name]">`
Footer: `Generated by [org.name] PM System · [org.email]`
Fallback text (if logo fails): `[org.name]` in `--primary` colour

---

## Authority rule

- Tier 1 data (SOW, baseline) → SOW Reference section
- Tier 2 data (requirements, deliverables, decisions) → main sections
- Tier 3 data (comms, change-log) → What Changed and Last Sync only
- Never cite research/ in the dashboard

---

## Steps

1. Read `org.json` — extract brand variables
2. Read all other pre-read files
3. Determine version number (count dashboard-versions/ + 1)
4. Extract all data points per section
5. Generate dashboard.html:
   - Substitute org brand variables throughout
   - Populate all sections from project files
6. Save to project root
7. Report:

```
✓ Dashboard generated — v[N]
  Org: [org.name] · Colour: [org.primary]
  Review: open dashboard.html in browser

When ready: "publish dashboard"
```

## Financials — hidden by default, tap to reveal

Financials are populated from `reference/sow.md` Tier 1 data.
Hidden behind the 🔒 Show Financials button in the header.

**Placeholders to populate from sow.md:**

| Placeholder | Source |
|---|---|
| `{{FIN_FIXED_FEE}}` | Fixed fee total e.g. `$22,000` |
| `{{FIN_PAID}}` | Amount paid to date e.g. `$11,000` |
| `{{FIN_PAID_DATE}}` | Date of last payment e.g. `Paid 16 Mar 2025` |
| `{{FIN_OUTSTANDING}}` | Remaining balance e.g. `$11,000` |
| `{{FIN_DUE_DATE}}` | When outstanding is due e.g. `Due on delivery` |
| `{{FIN_BUCKET_USED}}` | Variable bucket hours used e.g. `12` |
| `{{FIN_BUCKET_MAX}}` | Variable bucket max hours e.g. `40` |
| `{{FIN_BUCKET_REMAINING}}` | Remaining hours e.g. `28` |
| `{{FIN_BUCKET_MAX_VALUE}}` | Bucket max dollar value e.g. `7,000` |
| `{{FIN_SCHED_1_PCT}}` | First payment % e.g. `50%` |
| `{{FIN_SCHED_1_AMOUNT}}` | First payment amount e.g. `$11,000` |
| `{{FIN_SCHED_1_TRIGGER}}` | When due e.g. `Upon signing` |
| `{{FIN_SCHED_1_STATUS_CLASS}}` | CSS class: `status-paid` / `status-pending` / `status-upcoming` |
| `{{FIN_SCHED_1_STATUS}}` | Label: `Paid` / `Pending` / `Upcoming` |
| `{{FIN_SCHED_2_PCT}}` | Second payment % |
| `{{FIN_SCHED_2_AMOUNT}}` | Second payment amount |
| `{{FIN_SCHED_2_TRIGGER}}` | When due e.g. `Upon delivery` |
| `{{FIN_SCHED_2_STATUS_CLASS}}` | CSS class |
| `{{FIN_SCHED_2_STATUS}}` | Label |

**Bucket fill bar percentage:**
Calculate: `(hours_used / hours_max) * 100`
Set as `data-pct` attribute on `#fin-bucket-fill` element.

**If no variable bucket in SOW:**
Hide the bucket section entirely — set `display:none` on `.fin-bucket`.

**If financials not yet in sow.md:**
Populate with `—` placeholders — never show `{{}}` to the client.
